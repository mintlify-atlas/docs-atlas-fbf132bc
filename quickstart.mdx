This guide walks you through running StarDist nucleus detection on a brightfield H&E whole-slide image. By the end, you will have a working Groovy script that detects nuclei inside a selected annotation, adds shape and intensity measurements to each detected object, and cleans up GPU memory after inference.

## Prerequisites

Before you begin, make sure you have:

- **QuPath v0.5.0 or later** installed. See the [QuPath website](https://qupath.github.io) for downloads.
- **The StarDist extension installed.** Follow the [installation guide](/installation) if you haven't done this yet.
- **A pre-trained StarDist model.** For H&E images, download `he_heavy_augment.pb` (or another 3-channel RGB model) from [github.com/qupath/models](https://github.com/qupath/models). Check the credit and reuse information before downloading.
- **An H&E whole-slide image** open in a QuPath project, with at least one annotation drawn in the viewer.

## The H&E detection script

The script below is taken directly from the bundled `StarDistHE.groovy` template. You can also open it inside QuPath via **Extensions > StarDist > StarDist H&E nucleus detection script**.

```groovy StarDistHE.groovy
import qupath.ext.stardist.StarDist2D
import qupath.lib.scripting.QP
 
// IMPORTANT! Replace this with the path to your StarDist model
// that takes 3 channel RGB as input (e.g. he_heavy_augment.pb)
// You can find some at https://github.com/qupath/models
// (Check credit & reuse info before downloading)
def modelPath = "/path/to/model.pb"

// Customize how the StarDist detection should be applied
// Here some reasonable default options are specified
def stardist = StarDist2D
    .builder(modelPath)
    .normalizePercentiles(1, 99) // Percentile normalization
    .threshold(0.5)              // Probability (detection) threshold
    .pixelSize(0.5)              // Resolution for detection
    .measureShape()              // Add shape measurements
    .measureIntensity()          // Add nucleus measurements
    .build()
	 
// Define which objects will be used as the 'parents' for detection
// Use QP.getAnnotationObjects() if you want to use all annotations, rather than selected objects
def pathObjects = QP.getSelectedObjects()
 
// Run detection for the selected objects
def imageData = QP.getCurrentImageData()
if (pathObjects.isEmpty()) {
    QP.getLogger().error("No parent objects are selected!")
    return
}
stardist.detectObjects(imageData, pathObjects)
stardist.close() // This can help clean up & regain memory
println('Done!')
```

## Step-by-step walkthrough

<Steps>
  <Step title="Set the model path">
    Replace `"/path/to/model.pb"` with the absolute path to your downloaded model file on disk.

    ```groovy
    def modelPath = "/home/user/models/he_heavy_augment.pb"
    ```

    <Tip>
      As of extension v0.5.0, you can also specify just the model file name (without the full path) if the model is stored in a `stardist` or `models` subdirectory inside your QuPath user directory.
    </Tip>
  </Step>
  <Step title="Configure the builder">
    The `StarDist2D.builder(modelPath)` call returns a builder that lets you chain configuration methods before calling `.build()`.

    | Method | What it does |
    |---|---|
    | `.normalizePercentiles(1, 99)` | Normalizes pixel intensities between the 1st and 99th percentile of each tile, reducing the effect of staining variation |
    | `.threshold(0.5)` | Sets the minimum detection probability. Increase to detect only high-confidence nuclei; decrease to detect more nuclei with lower confidence |
    | `.pixelSize(0.5)` | Sets the resolution (in microns per pixel) at which detection runs. Should match the pixel size used to train your model |
    | `.measureShape()` | Adds nucleus shape measurements (area, perimeter, circularity, etc.) to each detected object |
    | `.measureIntensity()` | Adds per-channel intensity measurements within the detected nucleus region |
  </Step>
  <Step title="Select parent annotations">
    The script runs detection inside whatever objects are currently selected in the QuPath viewer.

    ```groovy
    def pathObjects = QP.getSelectedObjects()
    ```

    Select one or more annotations in the viewer before running the script. To run detection inside every annotation regardless of selection, replace this line with:

    ```groovy
    def pathObjects = QP.getAnnotationObjects()
    ```
  </Step>
  <Step title="Run detection">
    Open the QuPath **Script Editor** (**Automate > Script Editor**), paste the script (or open it via **Extensions > StarDist > StarDist H&E nucleus detection script**), then click **Run**.

    The script will:
    1. Check that at least one parent object is selected.
    2. Run StarDist inference tile-by-tile across each selected annotation.
    3. Add the detected nucleus objects as children of each parent annotation.
    4. Print `Done!` to the script log when complete.
  </Step>
  <Step title="Explore the results">
    When detection finishes, switch to the **Annotations** or **Hierarchy** tab in QuPath's left panel to see the detected objects. You can:

    - Click any detected nucleus in the viewer to inspect its measurements.
    - Open **Measure > Show detection measurements** to view a table of all nuclei and their shape and intensity values.
    - Export measurements via **File > Export measurements** for downstream analysis.
  </Step>
</Steps>

## GPU memory cleanup

The script calls `stardist.close()` after detection completes. This is important when running on a GPU — without it, CUDA memory may not be released until QuPath exits.

```groovy
stardist.detectObjects(imageData, pathObjects)
stardist.close() // This can help clean up & regain memory
```

<Warning>
  Always include `stardist.close()` at the end of your scripts when running on a GPU. Omitting it can cause CUDA out-of-memory errors if you run the script multiple times in the same QuPath session.
</Warning>

## Common adjustments

<Note>
  The values above are reasonable defaults for many H&E slides, but you will likely need to tune `threshold` and `pixelSize` for your specific model and tissue type. Refer to the documentation that accompanies your model for recommended settings.
</Note>

**Detecting too many or too few nuclei?** Adjust `.threshold()`. A value closer to `1.0` is more selective; a value closer to `0.0` detects more objects including low-confidence ones.

**Detections appear too large or too small?** Adjust `.pixelSize()` to match the pixel size (in microns) that the model was trained on. A mismatch here is a common source of poor results.

**Want cell boundaries, not just nuclei?** Add `.cellExpansion(5)` to grow nucleus detections outward and approximate cell boundaries. See the [fluorescence detection guide](/guides/fluorescence-detection) for an example.
